# /thread — Threads 內容引擎

唯一出口是 Threads。不產 LinkedIn、不產 blog 長文、不產 email(Michael 明確排除,2026-07-03)。

## 啟動時先讀(順序固定)
1. `docs/material-bank.md` — 真實素材庫。**個人故事的唯一合法來源。**
2. `docs/threads-log.md` 的「有效樣式」段 — 歷史成效學習。
3. `docs/story-thread-playbook.md` — 五拍公式全文(解剖、模板、反例)。

## 輸入模式(依 args 判斷,不確定就問一句)
| 模式 | 觸發 | 行為 |
|---|---|---|
| 關鍵詞 | `/thread 裁員潮` | 從素材庫找最能接住這個詞的真實故事切入 |
| 文章/新聞 | 貼全文或 URL | URL 用 WebFetch 抓;抽出「跟綠領轉職者有關的那一點」 |
| 雞湯反轉 | `/thread 雞湯 <那句話>` | 用一個真實故事把抽象安慰句打回具體 — 開場可引用該雞湯再反轉 |
| 舊 blog 文 | `/thread blog <標題>` | 對照 `docs/threads-from-blog.md` 既有 14 篇改寫 |
| 熱點雷達 | `/thread trend` | 派 1 個 subagent 掃本週台灣職涯/永續/就業市場高流量話題,回報 3 個+各附切角,**等 Michael 挑**,不直接產文 |
| 成效回填 | `/thread log <摘要> views=X reposts=Y clicks=Z` | 寫入 threads-log.md,並更新「有效樣式」段 |

## 產製管線(每篇都走,不可跳步)
1. **選角度** — 三框架擇一:身分故事(我也是)/ 被誤讀(明明很強)/ 反共識(你以為 X 其實 Y)。素材天然屬於哪個就用哪個,不硬套。
2. **套五拍脊椎**:場景 → 賭注 → 假動作 → 轉折 → 落點。冷開場第一行必須具體(時間/地點/對白/物件);禁止論點、鉤子問句、「在這個時代」。
3. **素材庫比對**:個人故事只能取自 material-bank.md「✅ 已確認」條目。用到「⬜ 待確認」→ 在產出頂端標 ⚠️。細節庫裡沒有 → 寫 `[待補真實細節:____]`,**絕不編造**。
4. **AI 味自檢**(全過才出稿,否則自動重寫):
   - 雞湯偵測:出現抽象安慰句(如「你不是不夠好」開頭)= 死
   - 工整排比偵測:三句以上同構 = 重寫
   - 凱旋結尾偵測:結尾是勝利宣言 = 改成安靜的身分句
   - 「不是…而是…」全文最多一次,只准放落點
   - 逐行檢查:不推進場景也不加深感覺的行 → 刪
5. **出稿格式(固定)**:
   - 3 個變體 + 一行排序理由(先發哪個、為什麼)
   - 每篇:主文 ≤500 字、斷行節奏排好;串接版標記斷點(斷在假動作前/轉折前)
   - 首則留言:接住主文同一條情緒線 + 一句產品橋接 + 連結
     `https://ahamoment-green-talent-os.vercel.app/zh-TW/mri?utm_source=threads&utm_medium=social&utm_content=<slug>`
   - 建議發文時段:故事 21:00–22:00 / 反共識 18:00–19:00 / 乾貨 22:30–23:30
6. **收尾提醒**(附在產出最後):發文後第一小時回覆每則留言;48 小時後回填 `/thread log`。

## 鐵律
- 全部 zh-TW。真實 > 漂亮;可以少發,不能造假。
- 連結永遠只放首則留言;主文不賣東西。
- 主文結尾是人際手勢(轉發給那個朋友/收藏),不是產品 CTA。
- 不自動發文:引擎產稿,Michael 按發送。
- KPI 不是產文量,是:每篇 MRI 導流數(utm clicks)+ Michael 每天花的分鐘數(目標 ≤5 分鐘)。
